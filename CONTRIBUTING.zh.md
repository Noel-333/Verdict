# 贡献指南

感谢你对 `Verdict` 的关注！我们欢迎每一位贡献者。请花几分钟阅读以下指南，让协作更顺畅。

## 2. 如何报告 Bug 或提出建议
- **检查现有 Issues**：先搜索是否已有相似问题，避免重复。
- **创建新 Issue**：请清晰描述复现步骤、预期结果和实际结果。
- **功能建议**：请描述使用场景和价值，而不是仅仅提出“我想要 X 功能”。

## 3. 如何提交代码（Pull Request 流程）
1. **Fork 仓库** 并克隆到本地。
2. **创建新分支**：请从 `main` 分支切出，分支名用 `feature/描述` 或 `fix/描述`。
5. **测试**：确保所有测试通过，并为新功能添加测试用例。运行 `[测试命令]`。
7. **同步最新代码**：提交前请 `rebase` 上游仓库的最新 `main` 分支，解决冲突。
8. **发起 Pull Request (PR)**：填写 PR 模板，描述改动内容、原因和测试情况。
